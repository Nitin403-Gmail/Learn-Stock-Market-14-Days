Module 07: Command Cookbook

Master the Terminal + AI integration. Learn to make Cursor execute development tasks automatically.

The Secret: Cursor can propose and run terminal commands. This turns "AI pair programmer" into "AI DevOps assistant".

1. Three Ways to Run Commands

Method When to Use Shortcut
Integrated Terminal You know the command. Just need to type it. Ctrl+`
Chat ‚Üí Command Generation You don't know the command syntax. Cmd+L (Chat)
Composer Agent Mode You want AI to run, see output, and iterate automatically. Cmd+I (Toggle Agent)

2. Practical Command Ideas (with Examples)

Scenario 1

Installing a NuGet Package

Manual Way: Search NuGet.org, copy the command, run it.

Cursor Way (Chat):

You: "What's the best logging library for .NET 8 and how do I install it?"

AI: "I recommend Serilog. Here's the command:"

dotnet add package Serilog
dotnet add package Serilog.Sinks.Console

üí° Pro Tip: Cursor will often show a "Run" button. You click it ‚Üí Terminal opens ‚Üí Command executes.

Scenario 2

Entity Framework Migrations

Chat Prompt: "I added a 'LastLoginDate' property to User.cs. Generate the migration."

AI Response:

dotnet ef migrations add AddLastLoginDateToUser
dotnet ef database update

Why this matters: You don't have to remember EF CLI syntax or migration naming conventions.

Scenario 3

Scaffolding Controllers

Prompt: "Generate an API Controller with CRUD operations for my Product model using scaffolding."

AI:

dotnet aspnet-codegenerator controller -name ProductsController \
  -async -api -m Product -dc AppDbContext -outDir Controllers

The AI not only suggests the command but can reference your @AppDbContext.cs to get the correct class name.

Scenario 4

Running Tests with Coverage

Prompt: "Run my xUnit tests and show me code coverage report."

dotnet test --collect:"XPlat Code Coverage"
dotnet tool install -g dotnet-reportgenerator-globaltool
reportgenerator -reports:"**/coverage.cobertura.xml" -targetdir:"coveragereport"
Scenario 5

Git Operations

Prompt: "Create a new feature branch called 'user-authentication' and push it."

git checkout -b feature/user-authentication
git push -u origin feature/user-authentication

Advanced: "Generate a commit message for my current staged changes about @UserController.cs"

3. Advanced: The "Agent Loop"

When Composer is in Agent Mode, it can:

  1. Run a command.
  2. Read the terminal output.
  3. If there's an error, fix the code and re-run.
Example Use Case:
You ask Composer: "Setup a Docker container for PostgreSQL and update my connection string."

What happens:
  • It writes a docker-compose.yml.
  • Runs docker-compose up -d.
  • If Docker isn't running, it tells you in the chat.
  • Updates appsettings.json connection string.

4. Command Cookbook Cheat Sheet

Goal Chat Prompt Expected Command
Update all NuGet packages "Update all packages to latest compatible versions" dotnet list package --outdated
dotnet add package [Name]
Check build errors "Build the solution and show me errors" dotnet build
Publish for production "Publish this API for Linux x64 production" dotnet publish -c Release -r linux-x64
Database seed data "Run the database seeder" dotnet run --seed (or custom script)
Format code "Format all C# files in this project" dotnet format
Clean build artifacts "Clean all bin and obj folders" dotnet clean

6. The "Script Generator" Strategy

Instead of asking for commands, ask for scripts.

Example: Deployment Script

Chat Prompt:

"Create a PowerShell script that builds my API, runs tests, and if tests pass, publishes to a 'dist' folder."

Result: AI creates deploy.ps1 with error handling, logging, and exit codes.

7. Key Takeaways

  • Never memorize CLI syntax again. Just describe what you want in natural language.
  • Use Agent Mode for multi-step command sequences (setup, run, validate).
  • Context matters: Reference @appsettings.json when asking about connection strings or environment variables.

Create a dev-tasks.md file in your repo with common commands. Then reference it: "Using @dev-tasks.md, run the database migration."
