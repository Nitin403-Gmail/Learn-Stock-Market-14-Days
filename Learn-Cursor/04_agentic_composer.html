Module 04: Agentic Workflows (Composer)

From "Coding Assistant" to "Coding Agent". Learn to build multi-file features in one shot.

graph TD A[Prompt: 'Build a Todo API'] --> B(Composer Agent); B --> C[Analyze Request]; C --> D{Plan Files}; D --> E[Create todo.cs]; D --> F[Create controller.cs]; D --> G[Update Program.cs]; G --> H[Show Diff to User]; H --> I[Apply All];

1. How Composer Works

Composer (Cmd+I / Ctrl+I) is a floating window separate from Chat. It has Write Access to your file system.

Note: Composer iterates. If you use "Agent" mode, it can run terminal commands, see the output, and fix its own errors.

2. Scaffolding a .NET Web API

This is where Composer shines. Instead of creating files one by one:

Example Prompt for Composer:

"Create a feature for 'Books'.
1. Create a `Book` model with Id, Title, Author.
2. Create a `BooksController` with full CRUD operations.
3. Update `Program.cs` to register any necessary services.
4. Ensure you use Dependency Injection.
5. Create a `IBookService` interface and implementation."

Outcome: Composer will create/edit all 4-5 files simultaneously. You just review and click "Accept All".

3. The "Broad Refactor" Workflow

Scenario: You want to change your DTO strategy across 50 files.

  • Bad Way: Open each file, edit, save.
  • Agentic Way: use Composer. "Iterate through the `Models` folder. For every class, create a matching DTO record in `Dtos` folder and add a mapping method."

4. Key Takeaways

  • Composer = Multi-File Creator. Use it when the task involves > 1 file.
  • Don't trust blindly. Composer can occasionally delete code you wanted to keep. checking the Diff is mandatory.
