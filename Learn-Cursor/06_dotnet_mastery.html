Module 06: .NET Mastery

Specific techniques for C#, Nuget, and Visual Studio migration.

Why Cursor for .NET?

Visual Studio has "IntelliSense", but Cursor has "Context Awareness". It understands how your Startup.cs affects your Controller.

1. Productive commands for .NET

Task Prompt Why use AI?
Nuget Install "Install the best library for PDF generation compatible with .NET 8." It suggests, compares, and runs dotnet add package.
EF Core Migration "I added a 'Phone' field to User.cs. Generate the EF Core Add-Migration command." Avoids memorizing CLI syntax.
Dependendy Injection "Register IUserService as Scoped in Program.cs." It finds the exact line in the builder chain.

2. Debugging workflows

Cursor cannot "attach" a debugger like Visual Studio (yet). But it is a better Diagnostic Tool.

The "Stack Trace Analysis"

  1. Run your app. It crashes.
  2. Copy the stack trace from the Terminal.
  3. Chat (Cmd+L): "Debug this exception." (Paste trace).
  4. Result: It will likely point out exactly which variable was null, often better than the VS Exception helper.

3. Dealing with `.sln` and `.csproj`

Cursor understands Solution files.

  • Multi-Project Search: @Codebase searches across all projects in the solution, not just the open one.
  • Project References: You can ask "Add a reference to the Shared project in the API project." and it will edit the .csproj file correctly.

4. Advanced: Converting Legacy Code

Scenario: Convert ASP.NET MVC 5 to Core

You have an old Global.asax.

Prompt: "Look at @Global.asax.cs. Translate this configuration into a modern .NET 8 Program.cs middleware pipeline."

It maps Application_Start to builder.Services... automatically.
