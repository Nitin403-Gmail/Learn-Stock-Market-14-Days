Module 03: Context Engineering

The art of preparing the "Prompt Payload". Garbage In, Garbage Out.

graph TD A[Prompt: 'Fix the bug'] --> B{Context Added?}; B -- No --> C[Generic Answer]; B -- Yes (@File A, @File B) --> D[Precise Solution]; style C fill:#f99,stroke:#333 style D fill:#9f9,stroke:#333

1. Precision vs. Recall tools

Symbol Function Best Use Case
@Files Reference File When you know exactly where the code is (e.g., @Program.cs).
@Codebase Semantic Search "Where in the project do we handle JWT Tokens?" (Exploration).
@Docs Documentation Use this for @MudBlazor or @Entity Framework Core.
@Git Git Context "Write a commit message for my current staged changes."

2. Advanced Technique: The .cursorrules File

You can create a file named .cursorrules in your solution root. The AI reads this before every prompt.

Example .cursorrules for .NET

# .NET Coding Standards
- Always use proper PascalCase for public members.
- Use 'var' only when the type is obvious from the right-hand side.
- Prefer 'async/await' over '.Result' or '.Wait()'.
- Use file-scoped namespaces (C# 10+).
- When using Entity Framework, always consider AsNoTracking() for read-only queries.

Putting this in your root folder saves you from correcting the AI on these points repeatedly.

3. Advanced: Debugging with Context

When you have an exception:

  1. Copy the Stack Trace.
  2. Open Chat.
  3. Type: "Analyze this stack trace looking at @MyController.cs and @MyService.cs".
  4. Paste the stack trace.

Why? The stack trace gives the symptom. The @Files give the cause context.
