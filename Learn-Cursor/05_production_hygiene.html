Module 05: Production Hygiene

Using AI to clean, document, and harden your code. Never write boilerplate again.

1. Documentation (XML Comments)

In .NET, XML comments (///) are gold standard but tedious to write.

  • Action: Highlight a method. Press Cmd+K.
  • Prompt: "Add detailed XML comments explaining the parameters and exceptions thrown."
  • Result: 
    /// <summary>
/// Calculates the total interest based on the principal amount.
/// </summary>
/// <param name="principal">The initial amount.</param>
/// <exception cref="ArgumentException">Thrown when principal is negative.</exception>
public decimal CalcInterest(decimal principal) { ... }

2. The Testing Machine (xUnit / NUnit)

Rule: Never write test boilerplate manually.

The "Break My Code" Technique

Instead of just asking for tests, ask the AI to find vulnerabilities.

Chat Prompt: "Look at @OrderService.cs. Identify 3 edge cases where this logic might fail, then write xUnit tests to cover them."

3. Refactoring for Performance

AI is surprisingly good at optimizing LINQ or suggest async implementations.

  • Prompt: "Refactor @MyLoop to use Parallel.ForEach for better performance."
  • Prompt: "Check if this SQL generation in EF Core causes N+1 problem."
