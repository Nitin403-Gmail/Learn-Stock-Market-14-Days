Module 01: The Cursor Mental Model

From Text Editor to Context Engine. Understanding the "Brain" behind the cursor.

1. The High-Level Roadmap

Real mastery of Cursor isn't about memorizing shortcuts; it's about managing Context and Intent.

2. The Core Concept: "Context is King"

Most developers treat AI tools like a Search Engine (Ask question ‚Üí Get answer).
Cursor Strategy: Treat AI like a Senior Pair Programmer who just joined the team 5 minutes ago.

The "5-Minute Colleague" Theory
If a new senior engineer sat next to you, they wouldn't know which file holds the auth logic or why you used a specific library 3 years ago. You must provide Context.
Approach Traditional Copilot Cursor Approach
Input "Write a function to sort list" "Look at @UserUtils.ts, use the sort logic there to sort @NewList"
Mechanism Text Prediction (Autocomplete) Context-Aware Diffing
Outcome Generic Code Project-Specific, Consistent Code

3. Why It Works (The "Why")

Cursor maintains a Vector Database of your local codebase.

  1. Indexing: When you open a folder, Cursor scans and indexes code chunks.
  2. Semantic Search: It understands "Authentication" even if you only search "Login".
  3. AST Awareness: It understands that C# Interface implementation requires implementing methods in the class.

4. Hands-On Exercise: Context Verification

Goal: Prove Cursor "sees" what you don't show it.
  1. Open any project.
  2. Create secret_data.txt: Write "The secret key is 9944-ALPHA."
  3. Create test.txt (Empty).
  4. Close secret_data.txt.
  5. Open test.txt, press Cmd+K / Ctrl+K.
  6. Type: "Write a sentence that includes the secret key from the text file."

Result: If indexed correctly, it will pull "9944-ALPHA" even from the closed file.

5. Key Takeaways

  • Cursor != VS Code + Plugin. It is a fork that indexes your file system.
  • Input = Intent + Context. Just "Intent" leads to hallucinations.
  • Codebase Awareness: The distinct power is accessing closed files via embeddings.
